悟流 / Stream of Wisdom 产品需求文档（PRD）
1. 项目概述

项目名称：悟流 / Stream of Wisdom  
GitHub 仓库：StreamOfWisdom  
使命：将 URL 输入的知识转化为简洁、生动的文本，如种子在心智中生根，而非沉重砖块。  
愿景：打造轻量、开源、社区驱动的平台，让用户轻松理解复杂知识。  
目标用户：知识工作者、教育者、学生、开发者，需将 URL 内容简化为高可读性文本。

2. 问题与机会

问题：URL 内容（如文章、网页）常冗长、抽象，难以快速理解。  
痛点：
信息复杂，缺乏清晰度和情感共鸣。
知识与用户已有理解脱节，难以提炼核心。
现有工具注重展示，而非有意义转化。


机会：通过 OpenAI 兼容模型将 URL 内容转化为简洁、生动的文本，提升理解效率。

3. 产品目标

核心目标：基于 URL 输入，调用 OpenAI 兼容模型，输出简洁、生动、高可读性的文本。  
具体目标：
简化复杂概念，保留深度和准确性。
融入情感共鸣，提升用户理解。
提供轻量开源平台，支持社区贡献。



4. 功能需求
4.1 核心功能

知识输入：
用户输入 URL（网页、文章），系统提取文本。


知识转化引擎：
使用 OpenAI 兼容模型（通过 API 调用，如 grok,qwen3）进行转化。
遵循以下提示词：

你是一个知识的转化者：

= 核心使命 = 将知识的形态重塑为大脑最渴望的样子。每个人的大脑都在寻找模式、需要意义。

= 转化原理 = 大脑不是硬盘，而是一个寻找模式、建立连接、创造意义的生命体。好的知识形态应该像种子而非砖块——能在心智中生根发芽。

= 价值指引 =
模式识别 > 细节记忆
与已知的连接 > 全新的信息
生动的画面 > 抽象的概念
情感的共鸣 > 理性的说服
可以行动的 > 只能知道的

= 转化方向 = 当收到任何文章或知识时：
找到其中最有生命力的核心洞察
将其转化为大脑天然亲近的形式
让复杂变简单，但不失深度
让抽象变具象，但不失准确

= 约束 =
不可歪曲核心要义。
不可捏造杜撰信息。
只给用户转化后的知识，不暴露你如何转化知识的框架
直接转化知识，不要暴露你是谁，你在干什么，你就是你

= 最终效果 = 读者应该感到如释重负。

prompt单独存放在文件中，不要硬编码到代码中

输出：简洁、生动的文本总结，融入类比和情感共鸣。


输出形式：
高可读性文本，突出核心洞察，使用生动语言。
无视觉化输出或行动指南。





个性化定制：
转化风格：叙事故事、技术总结。
复杂程度：初学者、中级。



4.2 技术需求

前端：
使用 Vanilla JavaScript（无 React），搭配 Tailwind CSS（CDN）构建轻量界面。
依赖通过 BootCDN 托管：<script src="https://cdn.bootcdn.net/ajax/libs/tailwindcss/3.4.1/tailwind.min.css"></script>


简洁的 DOM 操作，无复杂框架。


后端：
Node.js + Express，处理 URL 提取和 OpenAI API 调用。
使用 axios 提取 URL 文本（轻量 HTTP 客户端）。


数据处理：
OpenAI 兼容模型 API（如 GPT-4o mini）进行文本转化。
缓存提取的 URL 文本，减少 API 调用。


开源协议：MIT 协议。

5. 用户流程

用户访问 Web 平台，输入 URL。
系统提取 URL 文本，调用 OpenAI 兼容模型。
用户选择转化风格和复杂程度（默认“叙事故事”）。
系统返回简洁、生动的文本总结。
用户可编辑、保存或分享结果。
社区用户通过 GitHub 贡献提示词或模板。
